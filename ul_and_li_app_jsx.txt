const App = () => {
  const children = [
    <li key="1">
      <a href="https://www.google.com" target="_blank" rel="noopener noreferrer">
        <p>구글로 가세요</p>
      </a>
    </li>,
    <li key="2">
      <a href="https://www.instagram.com" target="_blank" rel="noopener noreferrer">
        <p>인스타로 가세요</p>
      </a>
    </li>,
    <li key="3">
      <a href="https://www.naver.com" target="_blank" rel="noopener noreferrer">
        <p>네이버로 가세요</p>
      </a>
    </li>
  ]

  return <ul>{children}</ul>
}

export default App


/*
//main.jsx에는
import ReactDOM from 'react-dom/client'
import App from './App'

ReactDOM.createRoot(document.getElementById('root')).render(
  <App />
)

*/


이 코드는 React 앱을 웹 페이지에 실제로 그려주는(렌더링하는) 역할을 하는 가장 핵심적인 시작 코드입니다. 한 줄씩 쉽게 풀어보면 👇


---

1️⃣ ReactDOM 불러오기

import ReactDOM from 'react-dom/client'

React를 브라우저(HTML)에 붙여주는 역할을 하는 라이브러리

client는 최신 React 방식 (React 18 이상)


👉 쉽게 말하면
➡️ "React를 화면에 출력할 수 있게 해주는 도구"


---

2️⃣ App 컴포넌트 불러오기

import App from './App'

App.jsx 또는 App.js에 있는 메인 컴포넌트

우리가 만든 UI의 시작점


👉 즉
➡️ "이걸 화면에 보여줄 거야!"


---

3️⃣ root 생성

ReactDOM.createRoot(document.getElementById('root'))

HTML 파일(index.html) 안에 있는


<div id="root"></div>

이 부분을 가져옴

👉 의미
➡️ "여기(root)에 React를 넣겠다"


---

4️⃣ 화면에 렌더링

.render(
  <App />
)

<App /> 컴포넌트를 실제 화면에 출력


👉 의미
➡️ "App을 root에 그려줘!"


---

🔥 전체 흐름 정리

이 코드는 한 문장으로 이렇게 이해하면 됩니다:

👉
"index.html의 root라는 공간에 App 컴포넌트를 렌더링해서 화면에 보여준다"


---

💡 비유로 이해하기

index.html → 빈 도화지

root → 그림 그릴 위치

App → 그림 내용

ReactDOM → 실제로 그림 그리는 사람



---

👍 한 줄 요약

👉 React 앱을 시작하고 화면에 출력하는 진입 코드


---

필요하면
👉 index.html 구조랑 같이 연결해서 더 쉽게 설명해줄게!
